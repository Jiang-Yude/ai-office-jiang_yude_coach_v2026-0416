# 這個老師適不適合我

## 為什麼需要這個技能包

每天有人推薦你新老師、新課程、新論文、新 Repo。你打開連結看半天，還是不知道「**值不值得花時間**」。

這個技能包用三節奏 + 五維度，幫你做出**個人化**的判斷。核心不是「這老師厲不厲害」，而是「**跟我現在的工作流合不合、學完我能拿來做什麼**」。

不是公評分卡。是專屬於你工作流的契合度檢查。

## 適用對象

| 主場景 | 副場景 |
|---|---|
| 老師、教練、講者、KOL、作者 | PDF、論文、文章 |
| 單堂課程（線上、實體、講座） | GitHub Repo |
| | 書、影片、Podcast |

主場景跟副場景共用同一套評估邏輯，只是輸入材料不同。

---

## 三節奏

### 🗺️ 節奏一｜建立你的學習地圖

**第一次使用時跑一次，之後可重新更新。**

技能包會問你 8 個問題（圍繞「**你的工作流**」），畫出你的學習地圖：

1. **主要工作 / 角色** — 一句話描述
2. **工作流 / 工作節奏** — 你的工作分哪幾個階段？
3. **目前最卡住的環節** — 哪一段是瓶頸？
4. **已掌握的技能** — 跟工作相關的列 5 個（用來「找鄰居」）
5. **想精進的環節** — 哪一段最想補強？
6. **學習偏好** — 看書／影片／一對一／實作／同儕
7. **時間預算** — 每週幾小時、能撐幾個月
8. **判斷準則** — 學完要能直接用？還是長線投資？時間 cap 多少？

存成 `my-learning-map.md`（在你本地，跟 SKILL.md 同層）。之後每次評估老師，AI 自動讀，不會重問。

詳細問答模板與範例見 [references/learning-map-template.md](references/learning-map-template.md)。

> **副產品**：這份學習地圖不只用來評估老師。它是你的工作流自我認識，也可以用來介紹自己給合作對象、跟新客戶開場、或定期回顧自己的成長軌跡。

### ⚡ 節奏二｜快速評估老師（不耗算力）

**輸入**（你提供）：
- 老師／資源的基本資料
- 主場景：老師簡介、課程簡介、課綱、講者背景
- 副場景：PDF 摘要、Repo README、論文摘要

**動作**：
1. 讀 `my-learning-map.md` 取得你的工作流脈絡
2. 比對輸入內容
3. 給出**契合度（20）+ 產出力（20）**兩個維度，共 40 分
4. 結尾固定問：「要不要進節奏 3A（深度可信度）或 3B（橫向對比）？」

**契合度的內部子項**借自江江原版的「定位四問」：
- 找鄰居：跟你已掌握的概念哪個最相關？
- 找邊界：差異在哪？
- 判斷價值：補了什麼缺口，還是重疊？
- 找落點：學了之後放哪裡用？

### 🔍 節奏三｜深度評估老師（要使用者確認、耗算力）

兩個子節奏，可單跑、可全跑：

#### 3A 可信度查

**觸發**：你選 A 並提供至少一個社群／網站連結。

**輸入**：
- 老師的社群帳號（Threads／FB／IG／X／YouTube／LinkedIn）
- 個人網站／部落格
- 過去作品／文章／影片

**動作**：
- 用 `web_search` 工具或專屬深度搜尋技能包（如 `web-deep-research`、`multi-ai-dispatcher`），三角度並行：
  - 官方介紹（他自己怎麼說）
  - 學員／讀者回饋（被教過的人怎麼說）
  - 業界口碑（同行怎麼說）
- 雙軸內評：**願景完整度**（想得清楚嗎）+ **實戰可信度**（做得出來嗎）
- 套用 FTC Coaching Scam 警訊清單做扣分檢查

**產出**：可信度（20）

#### 3B 橫向對比

**觸發**：你選 B。

**動作**：
1. 確認對比範圍：「找 2-3 個類似的[老師/課程/資源]比較，主軸是 X，OK 嗎？」
2. 用 `web_search` 或多代理搜尋找替代品
3. 表格對照：時間成本 / 金錢成本 / 適合誰 / 獨特性

**產出**：投資產出比（20）+ 不可替代性（20）

### 🎯 節奏結束｜合成最終報告

依「跑了哪些節奏」決定總分滿分：

| 已跑節奏 | 滿分 | 已評維度 |
|---|---|---|
| 只節奏 2 | 40 | 契合度、產出力 |
| 節奏 2 + 3A | 60 | + 可信度 |
| 節奏 2 + 3B | 80 | + 投資產出比、不可替代性 |
| 全跑 | 100 | 全部 5 維度 |

百分比建議：

| 百分比 | 建議 |
|---|---|
| 80% 以上 | 強烈推薦 |
| 60-79% | 推薦 |
| 40-59% | 條件性推薦／有保留 |
| 20-39% | 不太推薦 |
| < 20% | 略過 |

報告完整格式見 [references/scorecard-template.md](references/scorecard-template.md)。

## 五維度評分卡（總分 100）

| # | 維度 | 節奏 | 滿分 | 核心問句 |
|---|---|---|---|---|
| 1 | **契合度** | 2 | 20 | 跟我合不合？接得進我的工作流嗎？ |
| 2 | **產出力** | 2 | 20 | 學完能做出什麼？ |
| 3 | **可信度** | 3A | 20 | 這個老師可不可信？ |
| 4 | **投資產出比** | 3B | 20 | 划不划算？ |
| 5 | **不可替代性** | 3B | 20 | 有沒有更好的替代？ |

各維度評分標準、子項分數、扣分項清單詳見 [references/scorecard-template.md](references/scorecard-template.md)。

## 安裝

把這個資料夾放到你的 Claude 技能包目錄：

**Claude Code / Claude Desktop（Cowork 模式）：**
```
~/.claude/skills/teacher-fit-evaluator/
```

**自己的專案：**
```
<your-project>/_agent/skills/teacher-fit-evaluator/
```

重啟 Claude，技能包會在觸發詞出現時自動載入。

## 核心原則

1. **三節奏執行**：耗算力的節奏一定先確認再做，不自動燒
2. **個人化判斷**：吃 `my-learning-map.md`，這不是公評分卡，是「跟你合不合」
3. **不只給分數**：每個維度都要有「**哪裡能用、怎麼用**」的具體建議
4. **未查的維度誠實標 N/A**：不靠猜補分數
5. **學習地圖優先**：第一次使用時，先建立學習地圖再評估，順序不可逆

## 學習地圖更新

什麼時候該更新？
- 你的主要工作改了
- 工作流程做了大調整
- 之前的瓶頸解決了，有新瓶頸
- 三個月以上沒更新

更新方式：跟 AI 說「更新我的學習地圖」「我的工作改了」「重建學習地圖」。AI 會問你哪幾項要改，只更新有變的。

## 跟其他相關技能包配套

- 如果你有專屬的個人設定檔（個人簡介、價值觀、判斷標準），AI 會優先讀
- 如果你有深度搜尋工具（如 `web-deep-research`、`multi-ai-dispatcher`），AI 會在節奏 3A/3B 呼叫
- 沒有也沒關係，AI 用內建 `web_search` 也能跑

## 評估框架的學術／業界依據

五維度設計參考多個既有評估框架：

- **教育學界**：Kirkpatrick 四層、CIPP、Danielson Framework
- **產品評估**：Gartner Magic Quadrant、G2 Score
- **個人化匹配**：Multi-Criteria Recommender Systems、Global User Profile
- **學習決策**：CDC Quality Training Standards、ICF Coach Evaluation
- **可信度判斷**：SIFT 評估法、NIH Health Information Evaluation
- **風險訊號**：FTC Coaching Program Scam Warning（直接對應扣分項）

完整對照與來源詳見 [references/frameworks-reference.md](references/frameworks-reference.md)。

## 範例使用流程

**第一次使用**：

```
你：幫我看看這個老師 https://example.com/teacher-x

AI：我注意到你還沒建立學習地圖。先花 5 分鐘問你 8 個問題，
畫出你的工作流地圖，之後評估老師會更精準。
[1/8] 你目前的主要工作或角色是什麼？

你：（回答）

...（8 個問題）

AI：學習地圖建好了，存到 my-learning-map.md。現在開始評估這位老師...
```

**已建立過地圖**：

```
你：這個課值得上嗎？[貼課綱]

AI：（讀 my-learning-map.md）契合度 16/20、產出力 13/20，
建議「繼續深查」。要不要進節奏 3A（可信度）或 3B（橫向對比）？

你：A

AI：請給我老師的社群帳號或網站連結。

你：[貼連結]

AI：（搜尋三角度）可信度 14/20。累計 43/60。要不要再做節奏 3B？
```
